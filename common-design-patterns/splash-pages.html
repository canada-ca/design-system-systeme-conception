Splash pages: Alpha design pattern

Splash pages are where the user makes a language choice when landing on a bilingual domain or subdomain, such as cic.gc.ca or canada.ca.

When to use

Only use this pattern for a bilingual domain that has been approved for use. Ensure the domain respects the requirements for domains (link).

When to avoid

Do not use for unilingual domains - rather these should point to an equivalent landing page in each language.

Implementation

Always implement splash pages in a consistent fashion. For visual reference and to copy code, refer to the working example in the Web Experience Toolkit.

The splash page pattern includes the following:

  1. Background image
    A blurred landscape image. Default is provided, but may be customized.
  2. Government of Canada signature
    Always use the image file provided in the working example.
  3. Title
    Include organizational name or equivalent keywords in both official languages. Must concisely and accurately describe what you will find when clicking on the language link.
  4. Language links
    Include blue buttons labeled “English” and “Français”, each linking to an equivalent landing page.
  5. Terms and conditions links
    Include text links to English and French terms and conditions pages, labeled "Terms and conditions" and "Avis" respectively.
  6. Canada wordmark
    Always use the image file provided in the working example.

No additional elements or modifications permitted.
